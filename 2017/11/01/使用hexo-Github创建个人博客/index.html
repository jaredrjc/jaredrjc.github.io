使用hexo+Github创建个人博客

1.搭建环境

  • ubutnu 17.04
  • Git 2.14.1
  • node-v8.9.0
  • hexo-cli: 1.0.4
  • 安装Git(ubuntu)默认安装,可以更新
  • 安装 Node.js Node.js
  • 下载解压,进入解压目录的bin目录,打开终端输入
    ~$ npm install -g hexo
  • 进入到要创建博客的目录,执行(blog是存放hexo项目的文件夹)
    ~$ mkdir blog && cd blog

    2.创建blog

  • 执行
    ~$ hexo init

    或者直接在任意目录通过终端输入
    ~$ hexo init blog

初始化完成后可以使用
~$ hexo g && hexo s
在本地发布,当提示
INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.
说明hexo本地服务开启成功,通过
http://localhost:4000/
可以访问搭建完成的blog

3.部署blog

  • 进Github官网Github官网 登录
  • 点击 new repository 创建仓库
  • Repository name 的命名方式决定了你部署到github上之后能不能正常访问你的博客,命名格式为 user_name.github.io ,其中user_name为Repository name前显示的Owner
  • 创建完成后,会给出仓库的链接
  • 点击地址最后的复制按钮,复制仓库地址

  • 在blog根目录下找到 _config.yml 文件文本编辑器打开,找到 deploy: !
    修改为
    deploy:
    type: git
    repo: https://github.com/user_name/user_name.github.io.git

    其中user_name是你在github上创建仓库的时候写的名字(必须是自己github的用户名)

  • 在本地blog/的目录下执行
    hexo clean && hexo g && hexo d

    如果提示 Deployer not found: git
    则需要安装

等待重新清理,重新生成,然后部署到github上.提示
Username for ‘https://github.com‘:
输入上文提到的user_name然后根据提示输入邮箱,等待上传成功
INFO Deploy done:
然后通过user_name.github.io可以访问到已经部署的blog

4.遇到的坑

在Windows平台上,npm install hexo-deployer-git –save之后,会遇到一大堆的问题hexo-renderer-marked找不到或者没有安装或者需要 npm install hexo –save 之类
